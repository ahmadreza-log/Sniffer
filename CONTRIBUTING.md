# Contributing to Sniffer

Thanks for your interest in contributing.

## How to contribute

1. **Fork** the repository and clone your fork.
2. Create a **branch**: `git checkout -b feature/your-feature` or `fix/your-fix`.
3. Make your changes. Ensure the project builds:
   ```bash
   go build .
   ```
4. **Commit** with a clear message.
5. **Push** to your fork and open a **Pull Request** against the default branch.

## Code style

- Follow standard Go conventions (`gofmt`, `go vet`).
- Prefer clear names and short functions.

## Reporting issues

Use [GitHub Issues](https://github.com/YOUR_USERNAME/Sniffer/issues). For bugs, include:

- OS and Go version
- Steps to reproduce
- Expected vs actual behavior

## Pull requests

- Describe what changed and why.
- Keep PRs focused; one feature or fix per PR when possible.
